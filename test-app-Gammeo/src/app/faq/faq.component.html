FAQ

Questions Administration Système

Expliquer ce que va faire la commande suivante dans un système Unix :

$ cd ~/studiomagnetique

Change de dossier pour aller dans le répertoire studiomagnetique qui se trouve dans le répertoire de l'utilisateur en cours.

Sous Unix, quel est la signification du symbole ~ (tilde) ?

Le tilde représente le répertoire de l'utilisateur en cours plus communément appelé le répertoire 'HOME' de l'utilisateur.

Expliquer ce que va faire la commande suivante :

$ chmod -R application/src/ u+rwx,g+rx-w,o+rx-w;

Modifie les permissions d'accès au répertoire et tout ce qu'il contient. Le répertoire en question est src appartenant au dossier application. Les permissions ont été modifiées comme suit :
u+rwx : ajoute la lecture, l'écriture et l'exécution au propriétaire du fichier.
g+rx-w : ajoute la lecture, l'exécution et supprime l'écriture au groupe propriétaire du fichier.
o+rx-w : idem pour tous les autres utilisateurs.

Expliquer ce que va faire la commande suivante :

$ chown mysql:mysql /data/db

Change le propriétaire en mysql et le groupe propriétaire en mysql du fichier db qui est dans le dossier data.

Expliquer ce que va faire la commande suivante :

$ mkdir ~ test

Crée un dossier test dans le répertoire HOME de l'utilisateur en cours.

Questions Git

Qu'est-ce que git ?

Git est un contrôleur de versions. C'est un programme qui permet aux développeurs de conserver un historique des modifications et des versions de tous les fichiers.
Cela nous permet :
- travailler à plusieurs sans risquer de supprimer les modifications des autres collaborateurs;
- revenir en arrière en cas de problème;
- suivre l'évolution étape par étape d'un code source pour retenir les modifications effectuées sur chaque fichier.
Il va nous permettre de créer un dépôt local. Ce dépôt Git est un entrepôt virtuel de notre projet. Il permet d'enregistrer les versions de notre code et d'y accéder si besoin. L'intérêt de Git est donc le ssuivi des modifications mais aussi la sauvegarde des projets.

Que va faire la commande suivante :

$ git add

Le stage, désigne tous les fichiers modifiés que vous souhaitez voir apparaître dans votre prochain commit. C'est avec la fonction $ git add que l'on ajoute un instantané fichier au stage donc à la préparation pour le suivi de version.

Que va faire la commande suivante :

$ git checkout -b staging

Crée la branche (-b) staging et bascule (checkout) sur celle-ci.

Que va faire la commande suivante :

$ git rebase -i 796c088d6b303cc527540ae556515c818b9e9907

Cette commande permet de faire une rebase interactive sur le commit : 796c088d6b303cc527540ae556515c818b9e9907
Rebaser de manière interactive signifie que l'on a la possibilité de modifier les commits qui sont rebasés (les réorganiser, les supprimer). Cela donne un contrôle complet sur l'historique du projet. Il sert principalement à nettoyer son historique.

Que va faire la commande suivante :

$ git reset --hard origin/master

Supprime tout l'historique et les modifications effectuées sur ma branche actuelle pour qu'elle soit identique à la branche master.
